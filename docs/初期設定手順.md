# 初期設定手順

## 新規会社への導入時に必ず行う手順

### Step 1: 環境変数の設定

```bash
cd apps/portal
cp .env.example .env
```

`.env` を編集:

```env
# シークレットキー（必ず変更する）
NEXTAUTH_SECRET="node -e 'console.log(require(\"crypto\").randomBytes(32).toString(\"base64\"))'"
NEXTAUTH_URL="https://your-domain.com"   ← 実際のURLに変更

# 各アプリのURL（実際にデプロイしたURLに変更）
NEXT_PUBLIC_APP_WORK_URL="https://your-domain.com/work"
NEXT_PUBLIC_APP_INSPECTION_URL="https://your-domain.com/inspection"
NEXT_PUBLIC_APP_DASHBOARD_URL="https://your-domain.com/dashboard"
NEXT_PUBLIC_APP_CASE_URL="https://your-domain.com/case"
```

### Step 2: DB初期化

```bash
npm run db:generate
npm run db:push
```

### Step 3: 会社情報とアカウントの作成

`prisma/seed.ts` を編集して会社情報を設定:

```ts
const company = await prisma.company.upsert({
  where: { slug: '会社のスラッグ（英小文字）' },
  update: {},
  create: {
    name: '○○株式会社',       ← 正式会社名
    slug: 'company-slug',     ← 英小文字（URLに使用）
  },
})

// admin のメールアドレスと初期パスワードを変更
create: {
  email: '管理者のメールアドレス',
  name: '管理者の名前',
  passwordHash: await bcrypt.hash('初期パスワード', 12),
  role: 'admin',
  companyId: company.id,
},
```

```bash
npm run db:seed
```

### Step 4: アイコンの生成（本番用）

```bash
# ImageMagickがある場合
bash public/icons/generate-icons.sh

# ない場合はSVGのまま（Chrome/Edgeは対応）
# または: npx sharp-cli でPNG変換
```

### Step 5: ビルドと起動確認

```bash
npm run build
npm run start

# ブラウザで http://localhost:3010 を開いてログイン確認
```

### Step 6: 管理者への引き渡し

1. 管理者にURLと初期パスワードを伝える
2. 「管理者向け運用手順」を渡す
3. 最初のログイン後にパスワード変更をお願いする
4. スタッフアカウントの追加を管理者自身で行ってもらう

---

## チェックリスト

- [ ] `.env` の `NEXTAUTH_SECRET` は会社ごとに異なる値を使用
- [ ] `.env` の `NEXTAUTH_URL` は実際のドメインになっている
- [ ] 初期パスワードは `admin1234` などのデフォルト値のままにしない
- [ ] seedファイルのデモデータ（@demo.local）は本番環境に含めない
- [ ] DBバックアップの仕組みを確認済み
- [ ] 顧客引き渡しチェックリストを確認済み
