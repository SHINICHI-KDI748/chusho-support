# 今回未実装項目と将来拡張

## 今回未実装（PoC段階）

### 認証・セキュリティ
- MFA（TOTP）の実際の検証ロジック（DBフラグのみ準備済み）
- パスワードリセットメール送信（管理者手動対応）
- セッション一覧・強制ログアウト機能
- ブルートフォース対策（ログイン失敗回数制限）

### ユーザー管理UI
- ポータルからのユーザー新規作成フォーム（現在はAPI or seed）
- ユーザーのロール変更UI（現在はAPI直接）
- パスワード変更画面（ユーザー自身が変更できる画面）

### アプリ間連携
- アプリ間のセッション共有（現状は各アプリ別ログイン）
- 各アプリへの自動ログイン（SSO）

### 運用・監視
- エラーアラート通知（Slack/メール）
- ヘルスチェックエンドポイント（/api/health）
- 自動バックアップスクリプト

### その他
- 本番用アイコン（PNG形式）
- マルチテナントのサブドメイン自動振り分け
- 各アプリ側のロールチェック実装

## 技術的負債メモ

| 項目 | 現状 | 改善案 |
|------|------|--------|
| DBストレージ | SQLite (PoC) | PostgreSQL (本番) |
| アイコン | SVG | PNG 192/512px |
| セッション共有 | なし | 共通Cookieドメイン or JWT受け渡し |
| 各アプリの認証 | 独自（なし） | ポータルのJWTを検証 |
| エラーハンドリング | 最小限 | Sentryなど外部サービス |
