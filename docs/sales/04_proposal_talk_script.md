# 提案トークスクリプト｜断られにくい設計

---

## オープニング：警戒を解く

```
「今日は売り込みに来たわけじゃないので、
 まず現状を聞かせてもらえれば十分です。

 私が作っているのは、
 『大きなシステム』ではなくて、
 『今困っている1業務だけを小さく整理するアプリ』です。

 合わなければ使わなくていいし、
 1ヶ月試してダメなら解約できます。
 まずそれだけ伝えた上で話しましょう。」
```

---

## 課題の引き出し：ペインに触れる

```
「御社では、月末の集計作業ってどれくらい時間かかりますか？」

……（聞く）

「それって、誰かがExcelに転記してるんですか？
 それとも手書きの記録を見ながら？」

……（聞く）

「なるほど。
 それ、実はうちの実証先と同じ状況です。
 金属加工業の会社さんで、月末に2〜3時間かかってたんですが、
 アプリ1本入れたら15分以内になりました。
 転記ミスもゼロになって。」
```

---

## 提案：小さく始める論理

```
「今すぐ全部変える必要はないんです。

 まず1業務だけ試す。
 たとえば日報の入力だけ、とか、
 点検記録だけ、とか。

 それで『楽になった』と感じたら続ければいいし、
 『やっぱり紙のほうがいい』なら戻せばいい。

 月額15,000円から始められるので、
 1ヶ月試してやめても15,000円で済みます。

 大きなシステムを何百万円かけて入れるリスクとは
 まったく話が違います。」
```

---

## よくある断り文句への返し

### 「今は忙しくて…」

```
「わかります。
 だから、導入の手間も最小限にしています。
 初期設定は私がやります。
 現場スタッフへの説明も1時間以内で終わります。
 御社がやることは、最初の1〜2週間だけ
 試しに使ってもらうことだけです。」
```

### 「うちはIT苦手で…」

```
「スマホで入力するだけなので、
 紙の日報を書ける人なら使えます。
 導入時に一緒に説明しますし、
 マニュアルも渡します。
 ITが得意かどうかは関係ないです。」
```

### 「費用対効果がわからない…」

```
「だから最初は効果測定セットで入れます。
 導入前後の作業時間を一緒に計測して、
 『月何時間減った』を数字で出します。
 合わなければその時点で終わりにしていいです。」
```

### 「上に確認しないといけない…」

```
「もちろんです。
 確認に使える資料（事例シート・料金表）を
 すぐお送りします。
 上の方に見せてもらうとき、
 補足説明が必要であれば同席もします。」
```

 『試す価値ありそう』か『今は違う』か、
 どちらに近いですか？」

「ありがとうございます。
 では、1業務だけPoC提案書を作って送ります。
 費用と期間も入れますので、
 それを見てから判断してください。
 いつ頃までにお送りすれば都合いいですか？」
