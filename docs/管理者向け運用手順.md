# 管理者向け運用手順

## ユーザーの追加

1. ポータル（http://your-domain/）にadminでログイン
2. ホーム画面下の「👥 ユーザー管理」をクリック
3. 「+ 新規追加」ボタンを押す
4. 名前・メールアドレス・パスワード・権限を入力して保存

### 権限の選び方

| 権限 | 対象 | できること |
|------|------|-----------|
| 管理者 (admin) | 経営者・管理職 | 全機能 + ユーザー管理 + ログ閲覧 |
| 事務 (office) | 事務担当 | 作業入力・点検入力・ダッシュボード・事例生成 |
| 現場 (field) | 現場スタッフ | 作業入力・点検入力のみ |
| 閲覧 (viewer) | 外部確認者 | ダッシュボード閲覧のみ |

---

## ユーザーの無効化（退職・離職時）

1. 「👥 ユーザー管理」を開く
2. 該当ユーザーの行にある「無効化」ボタンを押す
3. 即座にログインできなくなる

**パスワードを変更するより無効化の方が確実です。**

---

## 監査ログの確認

1. ホーム画面下の「📋 監査ログ」をクリック
2. 誰がいつ何をしたか一覧で確認できる
3. 「CSV出力」ボタンでExcelに取り込める

### ログの種類

| 表示 | 意味 |
|------|------|
| ログイン | ログイン（成功・失敗） |
| 作成 | データの新規作成 |
| 更新 | データの変更 |
| 削除 | データの削除 |
| 管理操作 | ユーザー追加・変更など |
| エラー | システムエラー |

## バックアップの確認

### SQLite（PoC環境）

```bash
# バックアップファイルの確認
ls -lh apps/portal/prisma/dev.db
# → ファイルサイズと更新日時を確認
```

### バックアップを取る

```bash
cp apps/portal/prisma/dev.db backups/dev_$(date +%Y%m%d).db
```

---

## パスワードリセット（現在: 管理者が手動設定）

1. 「👥 ユーザー管理」→ 該当ユーザーの「編集」
2. 新しいパスワードを設定して保存
3. 本人に口頭またはメールで新パスワードを伝える
