# 30日実行プラン

## 目標
30日以内に、以下を達成する。

- ヒアリング5件以上
- アプリ候補10個
- MVP 2本以上
- 無料PoC 1件
- 事例シート 1枚
- 横展開先候補 10社

---

## 週ごとの目的

## Week 1：課題収集
### 目標
- 父親の会社で痛みを掘る
- 最初の狙いを1つ決める

### やること
- 父親と30分話す
- 現場 / 事務 / 管理者の3視点でヒアリング
- 痛み候補を10個出す
- スコアリングしてTop3を決める

### 成果物
- ヒアリングメモ
- アイデア一覧
- 優先順位表

---

## Week 2：最初のMVP作成
### 目標
- 1本目のMVPを作る
- 実際に触ってもらう

### やること
- 業務フローを1枚にまとめる
- 入力項目を削る
- Claude Code に実装指示
- 父親の会社で触ってもらう
- 不要機能を削る

### 成果物
- MVP v1
- 修正メモ
- 使いにくい点一覧

---

## Week 3：PoCと数字取得
### 目標
- 実運用で数字を取る

### やること
- 2週間程度使ってもらう
- Before / After を測る
- 時間削減、ミス削減、確認容易性を記録
- 必要なら軽修正

### 成果物
- 効果測定シート
- 利用者コメント
- 改善版MVP

---

## Week 4：横展開準備
### 目標
- 売れる形に変換する

### やること
- 事例シートを1枚作る
- LP文面を作る
- 紹介依頼文を作る
- 横展開先候補を整理
- 初回提案テンプレを作る

### 成果物
- 事例PDF
- LP原稿
- 営業文テンプレ
- 提案スライド簡易版

---

## 平日1時間の回し方

### 月
ヒアリング整理 / 仮説更新

### 火
要件定義 / 画面ラフ

### 水
Claude Code への実装指示

### 木
レビュー / 修正依頼

### 金
事例化 / 提案文 / 導線整備

### 土日
- 父親の会社で確認
- まとめて実装
- 次週の選定
- 提案資料整備

---

## 毎週レビュー項目

- 何の痛みを狙ったか
- 本当に困っていたか
- 説明は3分以内で通じるか
- 導入後の運用負荷は増えていないか
- 数字で効果を示せるか
- 同業へ横展開できるか
- 買い切り / 月額のどちらが適切か

---

## KPI

### 入力KPI
- ヒアリング件数
- 出したアイデア数
- MVP作成本数
- 父親の会社での確認回数

### 出力KPI
- PoC本数
- 実証済み事例数
- 紹介件数
- 商談件数
- 受注件数
- MRR / 単発売上

---

## Claude Code に投げる実装指示の型

### テンプレ
- 解決したい業務
- 現在の運用フロー
- 誰が使うか
- 絶対必要な機能3つ
- 不要な機能
- 入力項目
- 管理画面で見たいもの
- 1週間で作れる範囲
- デザイン優先度は低い
- ローカル or 簡易デプロイ前提
- 実証用なので拡張性より速度を優先
