# ロール設計表

## ロール一覧

| ロールID | 表示名 | 対象ユーザー | 説明 |
|---------|--------|------------|------|
| `admin` | 管理者 | 経営者・管理職 | 全機能にアクセス可能。ユーザー管理・監査ログ閲覧権限あり |
| `office` | 事務 | 事務担当者 | 入力・閲覧系全機能。ユーザー管理・ログ閲覧は不可 |
| `field` | 現場 | 現場スタッフ | 入力系のみ。集計・管理画面は非表示 |
| `viewer` | 閲覧 | 外部確認者・オーナー家族等 | ダッシュボード閲覧のみ |

## 実装方針

- ロールは `user.role` カラムに文字列で保存
- ポータル側は `lib/apps-config.ts` の `allowedRoles` でカード表示を制御
- 各アプリ側のアクセス制御は今後実装（現PoC段階では制御なし）
- Admin は将来的にMFA必須にする予定

---

## ロール追加手順（将来の拡張）

1. `lib/auth.ts` の `ROLES` 配列に追加
2. `lib/apps-config.ts` の各アプリの `allowedRoles` を更新
3. `prisma/schema.prisma` の `role` フィールドは文字列なのでスキーマ変更不要
4. ドキュメントを更新
