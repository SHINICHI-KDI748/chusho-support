# 現状PR

## 一言で何をやっているか
製造業の中小企業向けに、  
**紙・Excel・口頭で分散している業務を、小さな業務アプリに置き換え、父親の会社でPoC実証し、そのまま同業へ横展開できる形にするプロジェクト** を進めている。

---

## ここまでやったこと

### 1. 事業方針の固定
- 狙う市場を「製造業系の中小企業」と定義
- 実証先を父親の会社に固定
- 勝ち筋を「小さく強い痛み → 1週間MVP → 数字取得 → 事例化 → 横展開」と定義
- 目的を「月5万〜20万円の副収入化」に設定

### 2. 初期戦略の整理
- 初期に避けるべき領域を整理
  - 給与計算
  - 厳密な勤怠
  - 会計
  - 高リスク個人情報
  - 大規模SaaS
- 初期は「現場の面倒な確認・集計・伝達・見える化」に絞る方針を決定

### 3. アプリ候補の整理と優先順位付け
- 候補10個を整理
- Sランクとして以下を上位化
  1. Excel転記削減アプリ
  2. 点検チェックリストのスマホ化
  3. 現場日報の超簡略入力アプリ

### 4. PoC導線・営業導線の設計
- 父親の会社でヒアリング
- 1テーマに絞ってMVP作成
- 無料PoCで数字取得
- 事例化
- 紹介横展開
という導線を固定

### 5. 実装済みアプリ
#### アプリ1
**作業実績入力アプリ**
- 現場が入力
- records.json に蓄積
- Excel転記削減のPoC基盤

#### アプリ2
**点検チェックリストアプリ**
- 点検入力
- inspection-records.json に蓄積
- 未実施 / NGの見える化の基盤

#### アプリ3
**業務統合ダッシュボード**
- アプリ1とアプリ2のデータを1画面で確認
- 作業件数、点検件数、未実施、NG、要確認を表示
- 担当者別サマリ
- 詳細一覧
- CSV一括出力
- 管理者が毎朝3分で現場全体を把握できるPoCを実装済み

---

## 現状の価値
現時点で、単発の便利ツールではなく、以下の流れができている。

- 現場入力
- 点検入力
- 管理者向け統合把握

つまり、
**現場 → 管理**
までは、すでに1本の価値線として成立している。

---

## 次に作るもの
### アプリ4
**PoC効果測定・事例化アプリ**

役割は、
- Before / Afterの数字記録
- 管理者満足度 / 継続利用意向の記録
- 事例シート生成
- 営業文生成
- 紹介文生成

つまり、
**PoC → 数字取得 → 事例化 → 営業転用**
を閉じるためのアプリ。
