# 点検チェックリスト スマホ化アプリ v1.1

**「紙の点検表をスマホ入力に置き換え、記入漏れ・確認の手間・履歴管理を改善するアプリ」**

---

## クイックスタート

```bash
cd inspection-poc
npm install
npm run dev
# → http://localhost:3000
```

初回起動時に「機械設備 日常点検」「出荷前品質確認」のサンプルマスタが自動生成されます。

---

## 画面一覧

| URL | 主な機能 | 利用者 |
|-----|---------|--------|
| `/` | 点検入力（スマホ最優先） | 現場担当 |
| `/admin` | 今日の実施状況・一覧・記録編集・CSV | 管理者・事務 |
| `/master` | 点検対象・頻度・項目のUI管理 | 管理者 |

## 主要機能

### 点検入力（`/`）
- 日付・担当者・点検対象を選択
- 項目ごとに「○正常 / ×NG / —未実施」をタップ
- NG項目にはコメント欄が出現
- 写真添付（スマホはカメラ直接起動）— 自動でリサイズして保存
- 「点検結果を提出する」ボタンで登録

**目安：1件1〜2分**

### 管理画面（`/admin`）
#### 今日の点検状況カード（最上部）
- 日次対象で今日未実施 → **赤「本日未実施」**
- 週次・月次対象で未実施 → **黄「未実施」**
- 実施済み → **緑「実施済」＋件数**

管理者が朝イチでこのカードを見るだけで把握できる。

#### 記録一覧と編集
- 日付・対象・NGのみで絞り込み
- 各レコードの「編集」ボタンから日付・担当者・項目結果・備考を修正可能
- NG項目は赤ハイライト表示

#### CSV出力
- 列: ID, 日付, 点検対象, 担当者, NG有無, **NG件数**, **写真有無**, 全体備考, 登録日時, [各項目], NG詳細, NGコメント

### マスタ管理（`/master`）
- 点検対象の追加・無効化
- 点検頻度の設定（**日次 / 週次 / 月次**）— ドロップダウンから即時変更可能
- 点検項目の追加・並び替え・名前変更・削除
- 変更はすぐに入力フォームと管理画面の状況判定に反映

---

## 点検頻度の動作

| 頻度 | 「実施済」の判定基準 |
|------|------------------|
| 日次 | 今日の記録が1件以上ある |
| 週次 | 今週月曜〜今日の記録が1件以上ある |
| 月次 | 今月1日〜今日の記録が1件以上ある |

---

## PoC運用の進め方

### 導入前（Before測定）
1. 1週間の点検記録時間を計測（分）
2. 記入漏れ件数を集計
3. 管理者の確認時間を計測

### PoC期間（1〜2週間）
1. `/master` で実際の点検対象・頻度・項目を設定
2. 現場担当が `/` から毎回入力
3. 管理者が毎朝 `/admin` の「今日の点検状況」カードで確認
4. 週次で CSV 出力して比較

### 効果測定（After記録）
| 測定項目 | Before | After |
|---------|--------|-------|
| 点検記録時間（分/回） | | |
| 記入漏れ件数（週） | | |
| 管理者確認時間（分/日） | | |
| 担当者満足度（1〜5点） | | |

## 既存データのマイグレーション

`masters.json` に `frequency` フィールドがない場合（v1.0以前）、
起動時に自動で `"daily"` として補完されます。データ手動変換は不要です。

## 次の統合方針（Excel転記削減アプリとの連携）

App1（Excel転記削減）とApp2（点検チェックリスト）を統合した
**現場業務ダッシュボード**を将来的に構築する。

### 統合時のデータ設計
- 共通の `workers`（担当者マスタ）を持つ
- 共通の `date`（日付）をキーに両アプリのデータを結合
- ダッシュボード画面：「今日の作業記録 N件 / 点検 N件（未実施 N件）」

### 実装優先度
1. 担当者マスタの共通化
2. 日付別サマリーAPI（両アプリのデータを集約）
3. トップページのダッシュボード化
