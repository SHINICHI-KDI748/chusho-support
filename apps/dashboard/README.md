# 業務統合ダッシュボード v1.1

製造業の中小企業向け管理者ダッシュボード。
作業実績アプリ（App1）と点検チェックリストアプリ（App2）のデータを1画面に統合し、
**毎朝3分で現場全体の状況を把握**できる。

---

## このアプリができること

| 機能 | 説明 |
|------|------|
| 統合サマリー | 作業件数・点検件数・未実施・NG・備考ありを1画面で確認 |
| 要対応バナー | 未実施点検またはNGがある日は画面上部に赤バナーで警告 |
| 日別トレンドグラフ | 直近30日の作業件数・点検件数・NG件数を棒グラフ＋折れ線で可視化 |
| 工程別/対象別集計 | 工程名ごとの作業件数・点検対象ごとのNG率をタブ切り替えで確認 |
| 担当者別サマリ | 全担当者の作業件数・点検件数・NG件数を横断比較 |
| 期間フィルタ | 今日・昨日・直近7日・カスタム期間での絞り込み |
| 詳細一覧 | 作業実績・点検記録のタブ切り替え・キーワード検索・NGのみ抽出 |
| CSV一括出力 | 作業実績CSV・点検記録CSV・未実施一覧CSV（Excel対応BOM付き） |
| PoC評価記録 | Before/After の実測値を入力・保存・CSV出力できる専用ページ |

---

## できないこと（既知の制約）

| 制約 | 内容 |
|------|------|
| データ書き込み | ダッシュボードはRead Only。編集は各アプリの管理画面から行う |
| 写真表示 | App2の写真は別ポートのPublicに置かれているため非対応 |
| リアルタイム更新 | ページ再読み込みで最新データを取得する設計 |
| 複数PC同時書き込み | PoC単一ユーザー想定。競合対策は未実装 |

## システム構成

```
[App1: 作業実績入力]     [App2: 点検入力]
  localhost:3000           localhost:3001
        ↓                        ↓
  records.json       inspection-records.json
        ↓                        ↓
      [ダッシュボード: localhost:3002]
         /dashboard   — メイン画面
         /records     — 詳細一覧
         /poc-evaluation — PoC評価記録

[担当者共通マスタ]
  ../shared/workers.json
  ← アプリ1・2・ダッシュボード共通で参照
```

## セットアップ（初回）

### 前提条件
- Node.js 18以上がインストールされていること
- macOS / Windows（WSL）

### 手順

```bash
# 1. 依存関係を全アプリに一括インストール
cd small_company_support
npm run install:all

# 2. 3アプリを一括起動
npm run dev
```

これだけで以下の3アプリが同時起動します：
- http://localhost:3000 — 作業実績 入力/管理
- http://localhost:3001 — 点検チェックリスト 入力/管理
- http://localhost:3002 — 統合ダッシュボード（メイン）

### 別々に起動する場合

```bash
# ダッシュボードのみ
npm run dev:dashboard

# 作業実績アプリのみ
npm run dev:app1

# 点検アプリのみ
npm run dev:app2
```

## 担当者マスタの更新

`shared/workers.json` を編集して担当者を追加・更新します。

```json
{
  "workers": [
    {
      "id": 1,
      "name": "田中",
      "active": true,
      "aliases": ["田中さん", "TANAKA"]
    }
  ]
}
```

- `name` : 正規名（ダッシュボードに表示される名前）
- `aliases` : 表記ゆれのリスト。いずれかが一致すれば正規名に統一される
- `active: false` にすると非表示になる

## 他社導入時の前提条件

| 項目 | 内容 |
|------|------|
| 必要なアプリ | App1（作業実績）とApp2（点検）の両方が稼働していること |
| ポート | 3000・3001・3002 の3ポートが空いていること |
| PC | Mac または Windows（WSL）に Node.js 18以上 |
| ネットワーク | 同一PC内で完結。クラウド不要 |
| データ保管 | JSONファイル（`records.json` / `inspection-records.json`） |

別PCから参照する場合は `.env.local` の `NEXT_PUBLIC_APP1_URL` / `NEXT_PUBLIC_APP2_URL` を変更してください。

## 画面一覧

### `/dashboard` — 統合ダッシュボード
- 要対応バナー（未実施・NGがある場合）
- サマリーカード（5種）
- 未実施点検 / NG一覧 / 要確認作業実績（3カラム）
- 日別トレンドグラフ（直近30日）
- 工程別/点検対象別 集計
- 担当者別サマリ
- CSV一括出力
- 各アプリへのリンク

### `/records` — 詳細一覧
- タブ切り替え（全て / 作業実績 / 点検記録）
- 日付範囲・キーワード・NGのみ フィルタ
- NG行赤背景・要確認行オレンジ背景
- CSV出力

### `/poc-evaluation` — PoC評価記録
- Before（導入前）数値入力
- After（導入後）数値入力・満足度スコア入力
- 効果サマリー自動計算
- CSV出力（事例化・提案書に転用可能）

## PoC評価で取れる数字

### Before（現状）
| 測定項目 | 測定方法 |
|----------|----------|
| 作業実績確認時間（分/日） | 実測 |
| 点検状況確認時間（分/日） | 実測 |
| 抜け漏れ発見までの時間（分） | 実測 |
| NG確認の手間（ステップ数） | 実測 |
| 日次確認に使う画面/ファイル数 | カウント |

### After（このダッシュボード導入後）
| 測定項目 | 目標値 |
|----------|--------|
| 統合画面確認時間（分/日） | 3分以内 |
| 未実施発見時間（秒） | 10秒以内 |
| NG把握時間（秒） | 10秒以内 |
| 管理者満足度（5点満点） | 4点以上 |
| 継続利用意向（5点満点） | 4点以上 |

記録は `/poc-evaluation` ページから入力・CSV出力できます。

## トラブル対応

### 「アプリが開かない」
→ `npm run dev`（ルートディレクトリで）が実行されているか確認。
   ターミナルを閉じると停止します。

### 「ダッシュボードにデータが表示されない」
→ App1（3000）とApp2（3001）が起動しているか確認。
   データは `../excel-reduction-poc/records.json` と `../inspection-poc/inspection-records.json` を直読みしています。

### 「iCloud Drive でnode_modulesが壊れる」
→ プロジェクトフォルダをDesktopではなく `/Users/ユーザー名/Projects/` などiCloud管理外に移動してください。
