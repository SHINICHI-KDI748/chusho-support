# PoC実施手順書 — Excel転記削減アプリ

## 概要

このドキュメントは「Excel転記削減アプリ v1.0」のPoC（概念実証）を
父親の会社で実施するための運用手順書です。

---

## PoC基本設計

| 項目 | 内容 |
|------|------|
| 目的 | 転記・集計の手間を数字で可視化し、Before/Afterを取得する |
| 対象業務 | （例）組立ラインの日次作業記録 ← ヒアリングで確定する |
| 対象者 | 現場担当 __ 名、管理者 __ 名、事務担当 __ 名 |
| PoC期間 | __ 週間（推奨：2週間）|
| 成功基準 | 転記時間が週あたり __ 分以上削減 |

---

## 事前準備チェックリスト

### 1. アプリ側の準備
- [ ] `npm run dev` でローカル起動確認
- [ ] `/master` で工程名を実際の会社の名称に更新
- [ ] `/master` で担当者名を実際の名前に更新
- [ ] `/` から1件テスト入力して `/admin` に反映されることを確認
- [ ] CSV出力してExcelで開けることを確認

### 2. Before数値の記録
- [ ] 1週間あたりの転記時間を計測・記録（`before_after_measurement_sheet.md`）
- [ ] 集計完了までの時間を記録
- [ ] 転記ミス・修正件数を記録（直近1週間）
- [ ] 担当者の不満度スコアを記録（1〜5点ヒアリング）

### 3. 導入説明（3分で終わる）
以下をそのまま口頭で伝える：

> 「今から使うアプリです。
> 毎回、紙に書いていた内容をこの画面で入力してください。
> （入力画面を見せる）
> 日付・工程・担当者名・品目・数量を選んで数字を入れて、
> 「登録する」を押すだけです。
> 管理者はこちら（管理画面）でいつでも一覧が見られます。
> 何かあれば遠慮なく言ってください。」

## 運用フロー（PoC中）

### 毎日
| 時間帯 | 誰が | 何をする |
|--------|------|---------|
| 作業中〜終業前 | 現場担当 | `/` から作業記録を入力 |
| 終業後 | 管理者 | `/admin` で当日の件数・数量を確認 |
| 翌朝 | 事務担当 | （必要なら）CSVを既存Excelに貼り付ける |

### 週次（PoC期間中）
- 週末に転記時間・集計時間を計測してシートに記録
- 担当者に満足度（1〜5点）をヒアリング
- 入力漏れ・ミス件数を集計

---

## トラブル対応

### 「アプリが開かない」
→ `npm run dev` を実行しているか確認。ターミナルが閉じていると落ちる。

### 「入力フォームに工程名が出ない」
→ `/master` で工程名が有効（active）になっているか確認。

### 「誤入力した」
→ `/admin` の該当行の「編集」ボタンで修正できる。

### 「データが消えた気がする」
→ `records.json` ファイルが存在するか確認。
   `ls *.json` コマンドで確認できる。

## PoC終了時の進め方

1. `before_after_measurement_sheet.md` のAfter欄を全て埋める
2. 担当者全員に最終アンケートを実施
3. `case_study_template.md` に結果を記入
4. 「使い続けたいか」を確認 → YES なら本番化検討
5. 数字と感想をもとに近隣同業者への紹介文を作成

---

## 横展開への道

PoCで取れた数字（削減時間・ミス削減・満足度）を使って、
以下の流れで次の案件につなげる：

1. 事例文作成（`case_study_template.md` を使う）
2. 同業者に「うちで効果が出た」と話す
3. 無料デモ → 有料導入（5〜12万円）
4. 保守契約（月0.5〜1万円）

PoC中から **数字と声を記録しておくこと** が最重要。
