# Excel転記削減アプリ PoC完成版 v1.0

**「現場入力を直接データ化し、事務のExcel転記と集計手間を減らす小型アプリ」**

製造現場の「紙→転記→Excel」サイクルを減らし、
入力・集計・確認をひとつの画面で完結させます。

---

## クイックスタート

```bash
cd excel-reduction-poc
npm install
npm run dev
```

ブラウザで `http://localhost:3000` を開く。

初回起動時に工程名・担当者名のサンプルマスタが自動生成されます。

---

## 画面一覧

| URL | 説明 | 主な利用者 |
|-----|------|----------|
| `/` | 現場入力フォーム | 現場担当 |
| `/admin` | 管理一覧・集計・CSV出力 | 管理者・事務 |
| `/master` | 工程名・担当者名のマスタ管理 | 管理者 |

---

## 入力画面（`/`）の使い方

1. 日付・工程名・担当者名をセレクトで選ぶ
2. 品目/作業対象と数量を入力する
3. 画面下の「登録する」ボタンを押す
4. 「✓ 登録しました」と表示されたら完了
5. 工程・担当者は引き継がれるので、続けて次の品目を入力できる

**目安：1件30〜60秒**

---

## 管理一覧（`/admin`）の使い方

### 絞り込み
- 日付「から」「まで」で期間指定
- キーワードで工程名・担当者・品目を横断検索
- 「絞り込む」ボタンで反映、「リセット」で解除

### サマリー確認
- 件数・合計数量がバーに表示される
- 「詳細を見る」で工程別・担当者別・日別の集計テーブルが開く

### レコード編集
1. 修正したい行の「編集」ボタンをクリック
2. 各フィールドが入力可能になる
3. 「保存」で確定 / 「取消」でキャンセル

### レコード削除
1. 削除したい行の「削除」ボタンをクリック
2. 確認メッセージが表示される
3. 「削除」で確定 / 「取消」でキャンセル

### CSV出力
- 「CSV出力」ボタンをクリックするとダウンロードが始まる
- ファイル名：`records_YYYY-MM-DD.csv`
- BOM付きUTF-8のためExcelで直接開いても文字化けしない

## マスタ管理（`/master`）の使い方

### 工程名・担当者名の追加
1. 該当のセクションの入力欄に名前を入力
2. 「追加」ボタンまたはEnterキー
3. すぐに入力フォームのドロップダウンに反映される

### 名前の変更
1. 変更したい項目の「編集」ボタンをクリック
2. 入力欄で名前を修正
3. 「保存」またはEnterキーで確定

### 項目の無効化（非表示）
- 「無効化」ボタンをクリック → 入力フォームのドロップダウンから消える
- 無効化済みのボタンをクリックすれば「有効化」できる

**データは `masters.json` に保存される。コード編集は不要。**

## PoC運用の進め方

### 準備（導入前）
1. 父親の会社の工程名・担当者名を `/master` でセットする
2. 現状の転記時間・集計時間・ミス件数を `docs/before_after_measurement_sheet.md` に記録
3. PoC対象業務・期間・担当者を `docs/poc_runbook.md` に明記

### 試用（1〜2週間）
1. 現場担当がスマホまたはPCで `/` から入力
2. 管理者が毎日 `/admin` で件数・数量を確認
3. 事務担当が週次で CSV 出力して既存Excel業務との比較を行う

### 効果測定
- `docs/before_after_measurement_sheet.md` のAfter欄を毎週記録
- 担当者に満足度（1〜5点）をヒアリング

### 終了後
- Before/Afterの数字を `docs/case_study_template.md` にまとめる
- 次案件への説明資料として使う

## バックアップ方法

データは `records.json` と `masters.json` の2ファイルに保存される。

```bash
# 手動バックアップ（推奨：週1回）
cp records.json records_backup_$(date +%Y%m%d).json
cp masters.json masters_backup_$(date +%Y%m%d).json
```

または CSV出力を定期的に取得しておくと、最悪の場合もデータを復元できる。

## 制約と未実装項目

| 項目 | 状況 |
|------|------|
| 認証・ログイン | なし（URL知っていれば誰でもアクセス可） |
| 本番クラウド化 | なし（ローカル起動前提） |
| リアルタイム更新 | なし（手動リロード） |
| 権限分離 | なし |
| 同時書き込み競合 | 無防備（1人ずつ順番に使う前提） |

---

## 次の拡張候補3つ

1. **本番デプロイ化（Railway / Render など）**
   社内ネットワーク外からもアクセス可能にする。スマホ入力が本格化する。

2. **簡易認証（パスワード保護）**
   URL共有のみで十分なPoC段階を終えたら、最低限のパスワード保護を追加する。

3. **週次メール・Slack通知**
   管理者に「今週の件数サマリー」を自動送信する。確認コストがゼロになる。
