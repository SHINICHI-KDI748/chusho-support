# アプリ1レビュー：Excel転記削減アプリ

## 結論
現状は **PoCとしては完成扱いでよい** 。
ただし、**販売可能なPoC完成版** としては未完成である。

つまり、
- **父親の会社で触ってもらうMVP** としては合格
- **PoCを回して数字を取り、次案件に横展開するための完成物** としてはあと一段必要

現状のREADMEから確認できる完成済み範囲は以下。
- 現場入力画面
- 管理一覧画面
- 日付/キーワード絞り込み
- CSV出力
- SQLite保存
- ローカル起動手順

一方で、README自身が未実装として明示している重要項目は以下。
- データ編集・削除なし
- マスタ管理画面なし
- スマホ最適化は最低限
- 本番デプロイ手順なし
- バックアップなし

このため、「動くPoC」ではあるが、「PoC実証を回しやすくする完成版」には未達である。

### 2. 父親の会社でPoCを回す実用度

- しかし、PoC中に必ず起きる「入力ミス」「担当者/工程変更」「利用実績の見える化」に弱い

### 3. 横展開可能性
**68 / 100**

理由:
- テーマ自体は横展開向き
- ただし現時点では「案件化のための説明素材」「効果測定設計」「導入テンプレ」が不足

### 4. 今このまま止めてよいか
**PoC開始なら yes / 売り物化なら no**

## 完成条件

### A. プロダクト条件
1. 現場が迷わず1分以内で1件登録できる
2. 管理者が当日分/期間指定の集計をすぐ確認できる
3. CSV出力が安定して使える
4. 入力ミスを管理者が修正できる
5. 工程名・担当者名をコード編集なしで変更できる
6. スマホで現場利用しても崩れない

### B. PoC運用条件
1. Before/Afterの測定項目が定義済み
2. PoCの対象業務・対象者・期間が定義済み
3. 使い方説明が3分で終わる
4. PoC終了後に事例化できるログが残る

### C. 販売条件
1. 「誰の何分を削るアプリか」を1文で言える
2. 導入前後の変化を1枚で示せる
3. 価格の出し方が決まっている
4. 次社に見せるLP/事例文の素がある

- グラフライブラリ追加は不要。テーブルまたは簡易表示でよい

- 入力画面をスマホ前提で調整
- ボタン、余白、入力導線、スクロール量を改善
