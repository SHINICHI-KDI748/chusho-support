# 勤怠管理システム — 営業デモ版

製造業の町工場・小規模工場向け **勤怠入力システムの営業デモ**です。
ブラウザだけで動作し、セットアップ不要でその場で見せられます。

---

## 起動方法

### 方法①：ファイルをダブルクリック（最速）
`index.html` をダブルクリックするだけです。
Chrome / Safari / Edge どれでも動きます。

### 方法②：ローカルサーバー経由（推奨）

```bash
# Python 3 の場合
cd 勤怠管理システム
python3 -m http.server 8080

# Node.js の場合（npx利用）
npx serve .
```

ブラウザで `http://localhost:8080` を開きます。

## デモ操作手順

### 起動直後の状態
- 社員6名（田中・鈴木・佐藤・山田・伊藤・渡辺）
- 2026年2月〜3月の勤怠データ約150件が入った状態で起動します
- 異常データ（打刻漏れ・長時間労働など）も含まれています

### デモシナリオ（営業で使う順序）

#### 1. 勤怠入力画面
> 「従業員はこの画面から入力します」

1. 従業員名を選んで日付・出退勤時刻を入力
2. 「保存する」をクリック → 画面上部に保存確認が表示
3. 同じ社員×同じ日を選ぶと既存データが自動ロード → 編集もできる

#### 2. 日次一覧画面
> 「管理者はここで今日の状況を一目で確認できます」

1. 日付フィルタで確認したい日を選択
2. テーブルで出勤・退勤・労働時間・残業時間が自動計算済み
3. 色付きバッジで「正常 / 退勤未入力 / 長時間労働」が一目でわかる

#### 3. 月次集計画面
> 「月末の給与計算に使う集計もワンクリックです」

1. 対象月を選んで「集計する」
2. 上部のサマリーカードで全体状況を即確認
3. 「CSVエクスポート」で社員別集計をダウンロード → Excelで加工可能

#### 4. 異常値一覧画面
> 「問題のあるデータだけ抽出して確認・修正できます」

1. 開くと打刻漏れ・長時間労働などが色分けで表示
2. 「異常種別」フィルタで絞り込み可能
3. 「修正」ボタンで入力画面へ直接ジャンプ

#### 5. CSV管理画面
> 「今使っているExcelから移行するときはここでCSVを取り込めます」

1. `sample_attendance.csv` をドラッグ＆ドロップ（またはファイル選択）
2. プレビューで内容確認 → 「取り込む」でデータ反映
3. 既存データを全エクスポートしてExcelに渡すことも可能

## CSVフォーマット仕様

インポート用CSVは以下の形式で作成してください。

| 列名 | 必須 | 形式・例 | 説明 |
|------|------|----------|------|
| `employee_id` | 必須 | `EMP001` | 従業員ID |
| `employee_name` | 必須 | `田中 太郎` | 従業員名 |
| `date` | 必須 | `2026-03-01` | 日付（YYYY-MM-DD） |
| `clock_in` | 任意 | `08:30` | 出勤時刻（HH:MM） |
| `clock_out` | 任意 | `17:30` | 退勤時刻（HH:MM） |
| `break_minutes` | 任意 | `60` | 休憩時間（分・整数） |
| `note` | 任意 | `残業対応` | 備考・メモ |

- 1行目はヘッダー行（必須）
- 文字コード：UTF-8 または Shift-JIS どちらも可
- 同じ `employee_id` × `date` のデータは上書きされます
- `sample_attendance.csv` を参考にしてください

## 集計ルール

| 項目 | 計算式 |
|------|--------|
| 労働時間 | （退勤 − 出勤） − 休憩時間 |
| 残業時間 | 労働時間 − 8時間（マイナスの場合は0） |
| 出勤日数 | 出勤時刻が入力されている日数 |
| 打刻漏れ | 出勤または退勤のどちらかが空のレコード数 |

## 異常値判定ルール

| 異常種別 | 判定条件 |
|----------|----------|
| 出勤未入力 | `clock_in` が空 |
| 退勤未入力 | `clock_out` が空 |
| 休憩未入力 | `break_minutes` が空または0（出退勤が揃っている場合） |
| 長時間労働 | 労働時間が10時間（600分）超 |
| 不自然な打刻 | `clock_out` ≤ `clock_in` |

---

## 注意事項

- データは **ブラウザのローカルストレージ** に保存されます
- ブラウザの「閲覧データを削除」を実行するとデータが消えます
- デモ用のため認証・権限管理は実装されていません
- 本番運用にはサーバーサイドへの移行を推奨します

## 画面説明（営業用）

### 勤怠入力画面
従業員が毎日の出退勤を入力する画面です。名前と日付を選ぶだけで過去データが自動表示されるので、修正も簡単です。

### 日次一覧画面
管理者が「今日は誰が出勤・退勤しているか」を一覧で確認できます。労働時間・残業時間はすべて自動計算。問題のある行は色で目立つように表示されます。

### 月次集計画面
月末の給与計算や勤怠報告に使う集計画面です。社員ごとの総労働時間・残業時間・打刻漏れ件数を1クリックで集計し、CSVエクスポートできます。

### 異常値一覧画面
打刻漏れや長時間労働など、問題のあるデータだけを自動抽出して表示します。管理者が毎朝チェックするだけで見落としを防げます。

### CSV管理画面
今まで使っていたExcel/CSVファイルをそのまま取り込めます。逆に現在のデータをCSVで出力してExcelに渡すこともできます。既存運用との橋渡しができます。
